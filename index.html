  • 공통 컴포넌트를 재설계하고 기능별 모듈화

    • 기존 코드에서 기능과 스타일의 강한 결합과 의존성 문제로 유지보수가 어렵고, 작은 변경에도 공수가 증가

      -> 컴포넌트를 기능별로 모듈화하고 의존성을 제거해 독립성과 재사용성을 확보하고 각 페이지를 템플릿화하여 공통 구조를 재사용 가능하도록 개선해 개발 속도 향상.

  • 페이지 검색 환경 개선

    • 조회가 많은 관리 페이지가 1스텝만 데이터를 저장하는 구조로, 이전 상태 복원이 어려운 문제가 있었음.

      -> 히스토리를 스토어에 저장하던 레거시 구조를 URL 기반 상태 관리로 전환해 검색 히스토리를 관리하고, 여러 히스토리를 저장할 때 발생할 데이터 처리 이슈를 해결.

  • 번들링 개선

    • 기존 CRA환경에서 빌드와 개발 서버 실행 속도가 느려 작업 효율성이 저하되는 문제가 있었음.

      -> CRA가 React 공식 문서에서 제거된 점과 성능 한계를 고려해, 번들링 성능을 비교 테스트한 결과, Vite가 초기 빌드와 HMR에서 더 나은 성능을 보여 마이그레이션 진행.

      -> 빌드 및 개발 서버 실행 속도가 약 3배 빨라져 작업 속도와 개발 경험이 개선됨.
